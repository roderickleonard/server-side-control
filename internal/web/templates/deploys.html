Current behavior

  • If the target directory already contains a git repository, the panel fetches and fast-forward pulls the selected branch.
  • If the target directory is empty, the panel clones the repository directly into it.
  • The optional post-deploy command runs through `sh -lc` as the selected user.
  • Every successful deploy or rollback records the resulting commit SHA and previous commit in release history.
